# My Journey Learning React

When I first encountered React, I was already comfortable with HTML, CSS, and vanilla JavaScript. But React introduced a whole new way of thinking about building user interfaces.

## The Learning Curve

The concept of **components** took some getting used to. Breaking a UI into small, reusable pieces rather than thinking of a page as one big HTML file was a mindset shift.

Then came **state management**. Understanding how `useState` and `useEffect` work — and when to use each — was a key turning point.

## What Clicked for Me

The moment things started making sense was when I built my first full CRUD app. Creating, reading, updating, and deleting data through a React frontend backed by a REST API made everything feel real.

## Hooks Are Powerful

React Hooks changed the game for me. A few I use constantly:

- `useState` — Managing local component state
- `useEffect` — Running side effects like API calls
- `useContext` — Sharing state across components without prop-drilling
- `useReducer` — Managing complex state logic

## Tips for Beginners

1. **Build something real** — Don't just follow tutorials. Pick a project you care about.
2. **Understand JavaScript first** — React is JavaScript. Closures, promises, and array methods matter.
3. **Don't skip the docs** — The official React documentation is excellent.
4. **Practice state management** — It's the heart of React development.

## Looking Ahead

I'm now exploring Next.js and server-side rendering. The ecosystem keeps growing, and there's always something new to learn!
